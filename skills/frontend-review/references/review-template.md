# 审查模板

复制此结构生成 `docs/vX.Y.Z/review/<name>.md` 或 `review.md`

```markdown
# [页面/功能名] 代码审查

> 状态: DRAFT
> 版本: vX.Y.Z
> 来源: plans/<name>.md, test-report.md, progress.md
> 更新于: YYYY-MM-DD
> 失效原因:

## 结论

**通过** / **有条件通过** / **不通过**

简述理由（1–3 句）。

## 审查范围

| 项 | 路径/说明 |
|----|-----------|
| 需求 | prd/summarized/<name>.md |
| 方案 | design/<name>.md |
| 计划 | plans/<name>.md |
| 测试 | test-report.md |
| 变更文件 | （列表或 git diff 范围） |

## 问题清单

### 🔴 必须修复

| # | 位置 | 问题 | 建议 |
|---|------|------|------|
| 1 | `src/...` | | |

（无则写「无」）

### 🟡 建议

| # | 位置 | 问题 | 建议 |
|---|------|------|------|
| 1 | | | |

（无则写「无」）

### 🟢 可选

| # | 位置 | 问题 | 建议 |
|---|------|------|------|
| 1 | | | |

（无则写「无」）

## 维度核对

| 维度 | 结果 | 备注 |
|------|------|------|
| 需求符合 | 通过 / 有问题 | |
| 方案符合 | 通过 / 有问题 | |
| 范围符合 | 通过 / 有问题 | |
| 架构一致 | 通过 / 有问题 | |
| 代码风格 | 通过 / 有问题 | |
| 接口联调 | 通过 / 有问题 | |
| 正确性 | 通过 / 有问题 | |
| 可维护性 | 通过 / 有问题 | |
| 测试充分 | 通过 / 有问题 | |
| 安全 | 通过 / 有问题 | |
| 性能 | 通过 / 有问题 | |
| 回归风险 | 通过 / 有问题 | |

## 与 test-report 交叉

| test-report 项 | 审查结论 |
|----------------|----------|
| 未覆盖风险 | 是否接受 |
| 阻塞项 | 是否已解决 |

## 后续动作

- [ ] …
- [ ] 进入 `frontend-release`（仅无 🔴 时）

## 待确认问题

- …

（无则写「无」）
```

## 填写要点

| 章节 | 要求 |
|------|------|
| 结论 | 与 🔴 数量一致；有 🔴 必为「不通过」，具体参考 [/frontend-review] 的“结论判定”小节 |
| 问题清单 | 每条可定位、可执行 |
| 维度核对 | 快速扫盲，与问题清单不矛盾 |
| 后续动作 | 指明回步骤 4/5 或进入 release |
