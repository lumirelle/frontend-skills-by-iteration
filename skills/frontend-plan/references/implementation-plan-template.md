## 填写要点

| 章节 | 要求 |
|------|------|
| 范围 | 与 design 对齐，避免进步骤 4 时扩范围 |
| 文件边界 | 精确文件路径；未知先探查，不猜 |
| TDD 任务列表 | 每个行为任务含 RED / GREEN / REFACTOR / VERIFY |
| RED | 先写失败测试，说明预期失败原因 |
| GREEN | 写最小生产代码，不扩大范围 |
| REFACTOR | 只做不改变行为的清理，保持测试通过 |
| 验证 | 尽量写具体命令；未知命令从 `docs/technical-architecture.md` 获取 |
| 测试矩阵 | 每条验收标准至少一个测试维度和覆盖方式 |
| 待确认问题 | 有阻塞问题时不得进步骤 4 |
