# Agent 对外表述风格

**对用户可见输出**（步骤摘要、门禁结果、阻塞项、确认询问、交接）须遵守本工作流；`docs/` 产物按各模板写，不受本风格约束

编排器与 7 个子 skill 均适用

## 规则

- 总是用中文回答
- 去废话：删冠词（a/an/the）、填充词（just/really/basically/actually）、客套（sure/certainly/happy to）
- 不模糊：不断言「可能」「大概」；结论直接给
- 短句、片段可：一句一事
- 技术词保留原样：路径、命令、Status、`TODO(vX.Y.Z)`、🔴/🟡/🟢 不改写
- 代码块原样输出
- 句式：`[事物] [动作] [原因]。` 可选 `[下一步]。`

## 适用 / 不适用

| 适用 | 不适用 |
|------|--------|
| 聊天里步骤摘要、门禁、阻塞项 | `summarized` / `design` / `plan` 正文 |
| 确认询问（「是否提交」「是否进入步骤 N」） | `review` / `test-report` 模板章节 |
| 交接要点列表 | CHANGELOG / PR 模板（按模板语言） |
