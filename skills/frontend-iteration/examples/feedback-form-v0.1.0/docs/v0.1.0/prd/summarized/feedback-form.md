# 用户反馈表单

> 状态: ACTIVE
> 版本: v0.1.0
> 来源: prd/origin/feedback-form.md
> 更新于: 2026-06-27
> UI: 无
> 失效原因:

## 概述

已登录用户从「我的」→「意见反馈」进入表单，选类型、填内容、提交

## 页面与模块

| 模块 | 说明 | UI |
|------|------|-----|
| 反馈入口 | 「我的」增「意见反馈」，跳转表单 | — |
| 反馈表单 | 类型下拉 + 内容 + 提交 | — |

## 用户流程

1. 「我的」见入口 → 点击进入表单
2. 选类型（默认「功能建议」）→ 填内容
3. 提交 → 成功提示+清空；失败提示+保留

## 状态与交互

| 状态 | 触发 | 表现 |
|------|------|------|
| 默认 | 进入 | 类型默认「功能建议」；内容空；可提交 |
| 校验失败 | 空内容提交 | 「请填写反馈内容」；不发请求 |
| 提交中 | 发请求 | 按钮 disabled；「提交中…」 |
| 成功 | 接口成功 | 「反馈已提交」；表单重置 |
| 失败 | 失败/网络 | 「提交失败，请重试」；保留输入 |
| 未登录 | 未登录访问 | 既有拦截；不展示表单 |

**交互**：内容必填；trim 判空；提交中禁重复点击

## 数据与展示

| 字段 | 来源 | 规则 |
|------|------|------|
| 反馈类型 | 用户输入 | 功能建议/问题反馈/其他；默认功能建议 |
| 反馈内容 | 用户输入 | 必填；trim 非空 |
| 提交结果 | **新增 API**（待定） | 成功/失败驱动 UI；默认假设 `{ type, content }` |

## 平台差异

无（响应式跟项目断点）

## 边界情况

- 纯空白 → 校验不过
- 提交中重复点 → disabled，不二次请求
- 未登录 → 复用既有拦截；本迭代不新做鉴权

## 非目标

- 图片/附件
- 历史记录
- 内容长度上限
- 类型后台可配置

## 验收标准

- [ ] 「我的」有「意见反馈」入口，可进表单
- [ ] 类型默认「功能建议」，可切换
- [ ] 空/纯空白提交 → 「请填写反馈内容」，无请求
- [ ] 非空提交 → 提交中 disabled + 「提交中…」
- [ ] 成功 → 「反馈已提交」+ 重置
- [ ] 失败 → 「提交失败，请重试」+ 保留

- 未登录拦截已有；直接复用
