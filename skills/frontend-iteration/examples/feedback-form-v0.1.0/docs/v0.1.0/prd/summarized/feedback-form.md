## 概述

已登录用户在「我的」页面通过「意见反馈」入口进入反馈表单，选择反馈类型并填写内容后提交。

## 页面与模块

| 模块 | 说明 | 对应 UI |
|------|------|---------|
| 反馈入口 | 「我的」页面新增「意见反馈」入口项，点击跳转反馈表单 | — |
| 反馈表单 | 反馈类型下拉 + 反馈内容文本域 + 提交按钮 | — |

## 用户流程

1. 已登录用户进入「我的」页面 → 看到「意见反馈」入口。
2. 点击入口 → 进入反馈表单页面。
3. 选择反馈类型（默认「功能建议」）→ 填写反馈内容。
4. 点击「提交」→ 成功后提示「反馈已提交」并清空表单；失败后提示错误且保留已填内容。

## 状态与交互

| 状态 | 触发条件 | 界面表现 |
|------|----------|----------|
| 默认 | 进入表单 | 类型默认「功能建议」，内容为空，提交按钮可点击 |
| 校验失败 | 内容为空时点击提交 | 内容下方提示「请填写反馈内容」，不发起请求 |
| 提交中 | 发起提交请求 | 提交按钮 disabled 并显示「提交中…」 |
| 提交成功 | 接口返回成功 | 提示「反馈已提交」，表单重置为默认状态 |
| 提交失败 | 接口返回失败/网络错误 | 提示「提交失败，请重试」，保留已填内容，按钮恢复可点击 |
| 未登录 | 未登录用户访问表单 | 不展示表单，引导登录（沿用项目既有登录拦截） |

**交互说明**：反馈内容为必填，去除首尾空白后判空；提交进行中禁止重复点击。

## 数据与展示

| 字段/数据 | 来源 | 展示规则 |
|-----------|------|----------|
| 反馈类型 | 用户输入（下拉） | 选项：功能建议 / 问题反馈 / 其他；默认「功能建议」 |
| 反馈内容 | 用户输入（多行文本） | 必填，trim 后非空 |

## 平台差异

无（响应式适配沿用项目既有断点）。

## 边界情况

- 内容仅含空白字符 → 视为空，校验不通过。
- 提交过程中重复点击 → 按钮 disabled，不发起第二次请求。
- 未登录访问 → 沿用项目既有登录拦截，不在本迭代实现新的鉴权逻辑。

## 非目标

本次不做：

- 图片/附件上传。
- 反馈历史记录与列表。
- 反馈内容长度上限校验（origin 未提及，留待后续）。
- 反馈类型的后台可配置化。

## 验收标准

- [ ] 「我的」页面存在「意见反馈」入口，点击进入反馈表单。
- [ ] 反馈类型默认为「功能建议」，可切换为「问题反馈」「其他」。
- [ ] 反馈内容为空（含纯空白）时点击提交，展示「请填写反馈内容」且不发起请求。
- [ ] 内容非空时提交，提交中按钮 disabled 且显示「提交中…」。
- [ ] 提交成功后展示「反馈已提交」并重置表单。
- [ ] 提交失败后展示「提交失败，请重试」并保留已填内容。
