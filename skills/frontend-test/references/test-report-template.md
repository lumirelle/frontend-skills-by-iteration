# Test Report Template

复制此结构生成 `docs/vX.Y.Z/test-report.md`。

```markdown
# vX.Y.Z 测试报告

> 日期：YYYY-MM-DD | 范围：[页面/功能列表]

## 摘要

- 单元：通过 / 失败 / 未配置
- 集成：通过 / 失败 / 未配置
- E2E：通过 / 失败 / 未配置 / 手动验收
- 结论：**可进入 review** / **阻塞**

## 执行命令

| 层级 | 命令 | 结果 | 备注 |
|------|------|------|------|
| 单元 | `npm run test:unit` | exit 0 | |
| 集成 | `npm run test:integration` | exit 0 | |
| E2E | `npm run test:e2e` | exit 0 | |

## 验收标准覆盖

| 验收标准（来自 summarized） | 覆盖方式 | 对应用例/任务 | 结果 |
|----------------------------|----------|---------------|------|
| | 单元 / 集成 / E2E / 手动 | | 通过 |

## 补写测试

| 文件 | 说明 |
|------|------|
| | plan 矩阵要求、步骤 4 未覆盖 |

（无则写「无」）

## 手动验收

| 项 | 步骤 | 结果 |
|----|------|------|
| | | 通过 |

（无则写「无」）

## 未覆盖风险

- …

（无则写「无」）

## 阻塞项

- …

（无则写「无」）
```

## 填写要点

| 章节 | 要求 |
|------|------|
| 摘要 | 一眼可见能否进 review |
| 执行命令 | 写实际运行的完整命令 |
| 验收标准覆盖 | 每条 summarized 验收标准至少一行 |
| 未覆盖风险 | 诚实列出；有阻塞项则结论为「阻塞」 |
