# Summarized PRD 模板

复制此结构生成 `docs/vX.Y.Z/prd/summarized/<name>.md`。删除不适用的章节，不留空占位。

```markdown
# [页面/功能名]

> 状态: DRAFT
> 版本: vX.Y.Z
> 来源: prd/origin/<name>.md
> 更新于: YYYY-MM-DD
> UI: 有 / 无
> 失效原因:

## 概述

一句话说明本页/功能做什么、面向谁。

## 页面与模块

| 模块 | 说明 | 对应 UI |
|------|------|---------|
| | | `ui/xxx.*` 或 — |

## 用户流程

1. 用户进入 → …
2. 操作 → …
3. 结果 → …

## 状态与交互

| 状态 | 触发条件 | 界面表现 |
|------|----------|----------|
| 默认 | | |
| 加载中 | | |
| 空数据 | | |
| 错误 | | |
| 无权限 | | |

**交互说明**：（点击、跳转、表单校验、防抖/节流等）

## 数据与展示

| 字段/数据 | 来源 | 展示规则 |
|-----------|------|----------|
| | API / 本地 / 用户输入 | |

## 平台差异

（无则写「无」）

| 平台 | 差异 |
|------|------|
| | |

## 边界情况

- …

## 非目标

本次不做：

- …

## 验收标准

- [ ] …（可测试、可判定）
- [ ] …

## 假设

（从 origin 或 UI 推断、未经确认的内容）

- …

## 待确认问题

（需产品/设计确认）

- …
```

## 填写要点

| 章节 | 要求 |
|------|------|
| 概述 | 1–3 句，不重复 origin 全文 |
| 页面与模块 | 前端可拆组件的粒度 |
| 用户流程 | 主路径 + 关键分支 |
| 状态与交互 | 至少覆盖：默认、加载、空、错、权限（若适用） |
| 数据与展示 | 字段级；标明 API 来源（已有/新增/本地）；不写 API 实现细节 |
| 非目标 | 防止 scope creep |
| 验收标准 | 勾选框格式，一步一判 |
| 假设 vs 待确认问题 | 假设=你可先按此推进；待确认问题=必须确认 |
